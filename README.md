The first code, Palendrome.rs, is a code that takes an input from the user to check to see if the word is a palendrome. It then trims the input to a single word, for example if the text was "race car", it would change to "racecar". It would then check to see if the text is a palendrome, by iterating through the first half of the word (If the word is an odd number, it compares the median to itself). It checks to see if the respective letter from the end of the word is the same. Ex, if the second letter is being checked, it compares it to the value of the size of the word minus 2. It then prints if the code is a palindrome or not. 

For the second code, text-change.rs, it asks the user to input a text, to which it also simplifies the text. The user can then input the desirable function, whether to delete 5 random numbers, replace 5 random numbers, or add 5 random numbers. It then calls the respective function, which uses rng to chose a random value in the string to either delete, replace, or add to, and then returns the value

For the third code, authorization-check.rs, I made up some random usernames that were valid and invalid, based on a set of parameters that the username and password must follow. I then made the paramters of the username and password into code that checks the inputs, and returns whether or not the username and passwords were valid or not.
